# [PRD-FRONTEND] 프론트엔드 기술 상세 명세 (Frontend-Specific)

## 1. 개요
본 문서는 `PRD.md`에 정의된 유스케이스를 구현하기 위한 프론트엔드(`front-app`, `frontbeta-app`)의 기술적 상세 내용을 다룹니다.

## 2. 화면 구성 및 라우팅 (Screen Architecture)
- **Home (/front/)**: 상품 목록 탐색 및 장바구니 담기.
- **Cart (/front/cart)**: 장바구니 항목 편집 및 주문 프로세스 진입.
- **OrderList (/front/orders)**: 본인의 주문 이력 확인.
- **AddressManagement (/front/addresses)**: 배송지 목록 관리.
- **Login/Register (/front/login, /front/register)**: 인증 처리.

## 3. 기술 스택 및 구현 전략
- **프레임워크**: React + Vite + TypeScript.
- **API 연동**: Axios 기반의 중앙 집중형 API 클라이언트 구성.
- **디자인**: Vanilla CSS 활용, 기능 중심의 깔끔한 UI/UX 지향.
- **인증**: JWT 기반 인증 (LocalStorage 보관 및 Authorization 헤더 전송).

## 4. 기획 및 검증 전략 (Prototyping)
- **`frontbeta-app` (Prototype)**: 
    - Flyway 샘플 데이터를 하드코딩된 Mock 데이터로 활용하여 신규 유스케이스를 조기에 시각화함.
    - 배포 경로: `/study-start-java17/frontbeta/`
- **`front-app` (Main)**:
    - `frontbeta-app`에서 검증된 UI를 이관하고 실제 백엔드 API를 연동함.
    - 배포 경로: `/study-start-java17/front/`

## 5. 디자인 원칙 및 스타일 가이드
- **색상 정책**: 회색, 검은색, 흰색 등 무채색 위주로 구성하며, 포인트 컬러가 필요한 경우 **빨간색(#d00)**만 허용한다. 파란색 등 다른 유채색은 일절 사용하지 않는다.
- **UI 구성**: 복잡한 장식보다는 데이터의 정확한 표현과 사용자의 흐름에 집중한다. 모든 화면 데이터는 `PRD.md`의 유스케이스와 `specs/openapi/` 명세를 100% 준수한다.
- **모바일 최적화 (Responsive Design)**:
    - **가로 스크롤 방지**: 모바일 화면에서 테이블 형태의 데이터 나열은 가로 스크롤을 유발하므로 지양한다.
    - **카드 레이아웃 (Card-based Layout)**: 모바일(Breakpoint: 768px 미만)에서는 테이블 대신 정보를 수직으로 나열하는 카드 레이아웃을 적극 활용한다.
    - **상품 목록 최적화**: 
        - **상품 ID 노출**: 상품 ID는 상품 이미지의 윗줄에 배치한다.
        - **상품 ID 포맷팅**: 화면에 표시되는 상품 ID(목록, 상세 등)는 8자리 숫자로 고정하며, 8자리 미만일 경우 앞에 0을 채워서(Zero-padding) 표시한다. (예: ID 123 -> 00000123)
        - 상품명 영역은 2줄 고정 높이를 유지하며, 2줄을 초과할 경우 생략 부호(...)를 표시한다. 상품명 글자 크기는 정보 밀도를 높이기 위해 기본 스타일 대비 10% 축소하여 적용한다.
    - **장바구니 화면**:
        - **배송비 제외**: 장바구니 목록 및 결제 예정 금액 영역에서는 배송비를 표시하지 않으며, 상품 금액 합계만 노출한다. (배송비는 주문 생성 단계에서 결정됨)
    - **내비게이션**: 상단 메뉴바의 메뉴들은 우측 정렬을 기본으로 하며, 홈 아이콘 등 핵심 로고만 좌측에 배치한다.
    - **버튼**: 모바일에서는 주요 액션 버튼(주문하기, 검색 등)을 너비 100%로 설정하여 터치 편의성을 높인다.
