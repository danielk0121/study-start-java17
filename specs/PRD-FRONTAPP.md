# [PRD] 쇼핑몰 웹 프론트엔드 애플리케이션 (frontapp)

## 1. 제품 개요 (Product Overview)
본 문서는 `study-start-java17` MSA 백엔드와 연동되는 쇼핑몰 웹 애플리케이션의 화면 요구사항을 정의합니다. 사용자는 웹을 통해 상품을 탐색하고, 주문하며, 자신의 배송지를 관리할 수 있습니다.

- **기술 스택**: React, TypeScript, Vite
- **디자인 철학**: 
    - 이모지 및 화려한 꾸밈 요소는 화면에서 배제한다.
    - 폰트, 색상, 기본적인 여백 등 아주 간단한 요소만 사용하여 UI를 구성한다.
    - Vanilla CSS를 활용한 깔끔하고 기능 중심의 UI/UX를 지향한다.
- **주요 대상**: 상품 구매자(BUYER), 상품 관리자(MANAGER)

---

## 2. 화면별 요구사항 (Screen Requirements)

### 2.1. 공통 (Common)
- **네비게이션 바**: 로고, 상품 목록, 장바구니(아이템 개수 표시), 마이페이지, 로그인/로그아웃 버튼 포함.
- **권한 표기**: 회원이 로그인하면 우측 상단에 해당 회원의 권한(상품 구매자 또는 상품 관리자)이 명확히 표기된다.
- **권한 제어**: 로그인 상태 및 권한(BUYER/MANAGER)에 따라 메뉴 노출 제한.

### 2.2. 회원가입 및 로그인 (Auth)
- **구매자용**:
    - **회원가입 화면**: 이메일, 이름, 비밀번호 입력 필드. 가입 시 BUYER 권한 부여.
    - **로그인 화면**: 이메일, 비밀번호 입력 필드.
- **상품 관리자용**:
    - **관리자 회원가입 화면**: 이메일, 이름, 비밀번호, 관리자 인증 코드 입력 필드. 가입 시 MANAGER 권한 부여.
    - **관리자 로그인 화면**: 이메일, 비밀번호 입력 필드. 관리자 전용 진입점 제공.

### 2.3. 상품 탐색 (Product)
- **상품 목록(홈) 화면**: 전체 상품 리스트를 카드로 노출. 상품명, 가격, 브랜드, 카테고리 표시.
- **상품 상세 화면**: 상품의 상세 정보 확인 및 장바구니 담기/주문 수량 선택 버튼 제공.

### 2.4. 장바구니 (Cart)
- **장바구니 목록 화면**: 담긴 상품들의 리스트(상품명, 가격, 수량) 노출.
- **수량 조절**: 각 항목의 수량을 버튼으로 조절 가능.
- **금액 계산**: 각 항목별 합계 및 전체 주문 예정 금액 표시.
- **주문하기**: 장바구니 아이템들을 주문 생성 화면으로 전달.

### 2.5. 주문 및 배송 (Order & Shipping)
- **주문 생성(결제) 화면**: 선택한 상품 정보 확인, 등록된 배송지 중 하나를 선택하여 주문 진행.
- **주문 목록 화면**: 내가 주문한 내역(상품명, 상태, 배송지 주소, 날짜)을 리스트로 확인.

### 2.6. 마이페이지 (My Page)
- **회원 정보 관리**: 내 이름, 이메일, 권한 정보 확인.
- **배송지 관리 화면**: 배송지 목록 조회, 신규 배송지 등록, 기존 배송지 수정 및 삭제 기능 제공.

### 2.7. 관리자 대시보드 (Manager - MANAGER 권한 전용)
- **상품 관리 화면**: 상품 등록, 수정, 삭제 기능 제공.
- **통계 확인**: 상품별/기간별 판매 통계 차트 또는 표 제공.

---

## 3. 기술 구현 명세 (Frontend Technical Spec)
- **API 연동**: `specs/openapi/`에 정의된 스펙을 기반으로 Axios 또는 Fetch API를 사용하여 백엔드와 통신.
- **상태 관리**: React Context API 또는 가벼운 상태 관리 라이브러리 사용.
- **라우팅**: React Router를 활용한 클라이언트 사이드 라우팅(CSR).
- **인증**: JWT 기반 인증. 요청 헤더에 `Authorization: Bearer <token>` 부착.

---

## 4. 작업 프로세스 (Frontend Workflow)
1. **PRD 확인**: 본 문서를 기준으로 화면 설계 확인.
2. **API 연동 설계**: `specs/openapi/` 파일들을 참고하여 인터페이스(Type) 정의.
3. **컴포넌트 구현**: 원자 단위 컴포넌트부터 페이지 단위로 구현.
4. **검증**: 백엔드 서비스와 통합하여 시나리오 테스트 수행.
