# 제안서: 프로토타이핑 주도 개발 (Prototyping-Driven Development) 전략

## 1. 개요
PRD(제품 요구사항 문서) 기반의 개발 과정에서 발생할 수 있는 기획과 실 구현 간의 간극을 줄이고, 사용자 경험(UX)을 조기에 검증하기 위해 **프로토타이핑 주도 개발** 방식을 도입한다.

## 2. 핵심 관리 계층 (3-Layer Management)
프로젝트를 다음의 세 가지 계층으로 명확히 분리하여 관리한다.

1.  **기획 계층 (Planning/Prototyping - `frontbeta-app`)**
    *   목적: 샘플 데이터를 활용한 UI/UX 조기 검증 및 유스케이스 확정.
    *   산출물: 프로토타입 화면, 확정된 유스케이스, API 요구 데이터 모델.
2.  **프론트엔드 계층 (Frontend - `front-app`)**
    *   목적: 실제 서비스용 프론트엔드 구현 및 백엔드 API 연동.
    *   산출물: 실제 서비스 UI, API 연동 로직, 상태 관리.
3.  **백엔드 계층 (Backend - 각 마이크로서비스)**
    *   목적: 비즈니스 로직 구현 및 데이터 영속화.
    *   산출물: RESTful API, DB 스키마, 메시징 처리.

## 3. 작업 프로세스 (Workflow)
1.  **PRD 초안 작성**: `specs/PRD.md`에 핵심 요구사항 정의.
2.  **프로토타이핑 (`frontbeta-app`)**: 
    *   Flyway 샘플 데이터를 활용하여 UI를 빠르게 구성.
    *   실제 화면을 보며 유스케이스의 모순점이나 데이터 누락 확인.
3.  **API 규격 확정**: 프로토타입에서 도출된 데이터 모델을 기반으로 `specs/openapi/` 정의.
4.  **병렬 개발**:
    *   프론트엔드: `frontbeta-app`의 UI를 `front-app`으로 이관하고 실제 API 연동.
    *   백엔드: 확정된 API 규격에 따라 기능 구현.

## 4. 기대 효과
*   **재작업 감소**: 개발 완료 후 UI 수정으로 인한 백엔드 API 변경 최소화.
*   **커뮤니케이션 명확화**: 텍스트(PRD)가 아닌 화면(Prototype)을 기준으로 의사결정.
*   **품질 향상**: 실제 데이터와 유사한 샘플을 통해 엣지 케이스 조기 발견.
